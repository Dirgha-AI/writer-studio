# Getting help

Before opening an issue, please use the channel that matches your question.

## Documentation first

Most "how do I…" questions are answered in the docs. Quick links:

- [README](README.md) — what the project does and how to run it.
- [CONTRIBUTING](CONTRIBUTING.md) — how to make changes.
- [SECURITY](SECURITY.md) — how to report vulnerabilities.

## Where to ask

| I want to… | Go here |
|---|---|
| Ask "how do I…" or "does this support…" | [GitHub Discussions](https://github.com/dirghaai/writer-studio/discussions) |
| Share a tip, setup, or workflow | [Discussions — Show and tell](https://github.com/dirghaai/writer-studio/discussions/categories/show-and-tell) |
| Report a bug | [Open an issue → Bug report](https://github.com/dirghaai/writer-studio/issues/new?template=bug.md) |
| Request a feature | [Open an issue → Feature request](https://github.com/dirghaai/writer-studio/issues/new?template=feature.md) |
| Report a security vulnerability | See [SECURITY.md](SECURITY.md) — **do not** use public issues |
| Talk about commercial / enterprise use | `enterprise@dirgha.ai` |
| Press, partnerships, general questions | `hello@dirgha.ai` |

## Response times

These are targets, not guarantees, unless you have a commercial agreement.

- Security reports: acknowledged within 24 hours (business days).
- Bug reports: triaged within one week.
- Feature requests: reviewed at least monthly.
- Discussions: community-answered; maintainers chime in when they have time.
- Enterprise support: per your contract.

## Before you open an issue

- Search existing issues and Discussions first.
- Reproduce on the latest release.
- Include version, OS, runtime (Node / Bun / Deno) version, and any relevant environment.
- Redact API keys and any secrets from your logs.

## Commercial support

For on-premise deployment, custom integrations, SLAs, or indemnification, contact `enterprise@dirgha.ai`.
